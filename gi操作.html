本地分支与远程分之的关联操作
创建分支 git branch dev 切换分之 git checkout dev 创建并切换到该分支 git checkout -b dev 删除本地分支 git branch -d dev (不能当前分支) 删除远程分之 git push origin :dev 查看本地与远程的分支 git branch -av
查看本地与远程关联的分支 git branch -vv
1-本地分支推送到远程(本地有远程没有)
方法一（dev是分支名）
(1)-创建一个新分支并切换到该分之 git checkout -b dev
(2)-推送到远程 git push origin dev
方法二
(1)-创建一个新分支并切换到该分之 git checkout -b dev
(2)-推送到远程 git push --set-upstream origin dev
2-远程分支拉取到本地(远程有本地没有)
方法一（dev是分支名）
(1)-git pull
(2)-git checkout -b dev origin/dev #解释：意思是创建并切换到dev分支，并且dev分支会拉取远程上的dev分支的内容并关联
ps:(创建并切换的分支名必须和远程上面的分支名一样，才能拉取关联对应分支和对应分支的内容)
方法二（dev是分支名）
(1)-git pull
(2)-get checkout --track origin/dev #解释：在本创建一个dev分支,并且和远程上的dev分支相关联
3-冲突解决(远程与本地冲突 ) 3.1-git remote show origin #查看远程有没有新的提交 3.2-有冲突的话 (1)-git pull (2)-修改有冲突的文件 (3)-git add 冲突文件 (4)-git commit -m '注释' (5)-git push
