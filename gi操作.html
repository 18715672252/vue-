本地分支与远程分之的关联操作
创建分支 git branch dev 切换分之 git checkout dev 创建并切换到该分支 git checkout -b dev 删除本地分支 git branch -d dev (不能删除当前分支) 删除远程分之 git push origin :dev 查看本地与远程的分支 git branch -av
查看本地与远程关联的分支 git branch -vv
删除文件 git rm 文件名
文件重命名 git mv 1.txt 2.txt //把1.txt该名为2.txt
查看文件夹的所有文件 ls
命令行修改文件 1-vim 文件名 2-Insert切换编辑还是替换 3-左下角显示替换后按esc键 输入(1)-:wq保存并退出 (2)-:q!不保存直接退出
查看文件内容 cat 文件名
查看commit的差异 git diff HEAD HEAD~1 //查看最近一次提交与最近一次提交的上一次提交的差异
修改最近一次提交信息 git commit --amend 然后修改文件的过程
暂存区与工作区比较 git diff --cached
把所有文件恢复到和工作区一样 git reset HEAD
比较分支差异 git diff dev master
比较不同分支下相同文件名的文件的差异 git diff dev master -- 2.txt
删除文件 git rm 文件名 显示当前文件的路径 pwd .gitignore配置 *.d 任何以.d结尾的文件不被追踪 aa/ 任何以.aa结尾文件夹下文件不被追踪 node_modules文件夹不被追踪 1-本地分支推送到远程(本地有远程没有)
方法一（dev是分支名）
(1)-创建一个新分支并切换到该分之 git checkout -b dev
(2)-推送到远程 git push origin dev
方法二
(1)-创建一个新分支并切换到该分之 git checkout -b dev
(2)-推送到远程 git push --set-upstream origin dev
2-远程分支拉取到本地(远程有本地没有)
方法一（dev是分支名）
(1)-git pull
(2)-git checkout -b dev origin/dev #解释：意思是创建并切换到dev分支，并且dev分支会拉取远程上的dev分支的内容并关联
ps:(创建并切换的分支名必须和远程上面的分支名一样，才能拉取关联对应分支和对应分支的内容)
方法二（dev是分支名）
(1)-git pull
(2)-get checkout --track origin/dev #解释：在本创建一个dev分支,并且和远程上的dev分支相关联
3-冲突解决(远程与本地冲突) 3.1-git remote show origin #查看远程有没有新的提交 3.2-有冲突的话 (1)-git pull (2)-修改有冲突的文件 (3)-git add 冲突文件 (4)-git commit -m '注释' (5)-git push 4-标签 4.1-标签的作用 例子：已经发布了v1.0,v2.0两个版本 ， v3.0已经开发到一般 ， 现在想直接跳过v3.0的版本 ， 去开发v4.0版本(这个时候就用到了标签) 说明：新建一个tag(标签)是基于当前分支所指向的commit 1.新建一个标签-git tag -a 标签名 -m '标签备注';新建一个指向指定commit的标签-git tag -a 分支名 commitID 2.推送到远程-git push origin 标签名 3.查看某个标签-git show 标签名 4.查看本地所有标签-git tag 5.查看远程所有标签-git ls-remote --tags origin 6.删除本地标签-git -d 标签名 7.删除远程标签-git push origin :标签名 8.检出标签-git checkout -b 分支名 标签名(创建一个新分支指向要检出标签的所指向commit)
